Headers use ALL CAPS 11px at #6b7280. Body text 13px. Rows hover to #f8f9fc. Bordered with #e5e7eb. First column is always the primary identifier — rendered in #5865F2 as a clickable link using .td-link.

Guidelines · 16

Do's & Don'ts

Core rules for VX v2. Follow these to keep the UI consistent across all modules.

check_circleDo
  • Place KPI card icon tile in the top-right corner, never left of the value.
  • Use ALL CAPS 11px for data table column headers with 0.08em letter-spacing.
  • Use underline-style tabs (2px brand border-bottom) inside data panels.
  • Use the status pill style (tinted background + border + dot + label) consistently across all contexts.
  • Use breadcrumb in the page header — parent grey, current page bold.
  • Use module-specific icon tile colours consistently across sidebar, charts, and tables.
  • Use #5865F2 (brand) as the single primary accent — chart bars, active nav, focus rings, avatars.
  • Use #2DCE89 for positive monetary values and approved states.
cancelDon't
  • Don't use filled pill tabs inside table panels — underline only.
  • Don't use page-background colour (#f0f2f5) inside a card surface.
  • Don't mix button sizes in the same action row — pick one size.
  • Don't use ad-hoc hex values outside the defined token palette.
  • Don't display ALL CAPS names — always apply Title Case at render time.
  • Don't use a full-screen modal for a simple lookup — use a combobox dropdown.
  • Don't render status with only colour — always pair with a tinted background pill so the label is readable without relying on colour alone.
  • Don't use pagination when search is available.

Pattern · 19

Inline Status Banners

Contextual banners placed inside panels, forms, and filter builders. Four severity levels — Info, Warning, Error, Success — each with a defined colour, icon, and trigger condition.

Types & When to Use
info
Info — Simple filter mode. Switch to Advanced for grouping and nested logic.
Cyan · #11CDEF · When to use
  • Contextual guidance about the current mode or screen state.
  • Non-critical hints that help the user take the right next step.
  • Feature announcements or tips (e.g. "New policy effective 1 Apr").
warning_amber
Warning — Your session expires in 5 minutes. Save your work before being logged out.
Orange · #FB6340 · When to use
  • Something may go wrong but hasn't yet — the user can still act.
  • Approaching limits (session timeout, leave balance running low).
  • Conditions that could produce unexpected results (e.g. overlapping dates).
error_outline
Error — End date cannot be before start date. Please correct and resubmit.
Red · #F5365C · When to use
  • Action failed or cannot proceed — user must fix before continuing.
  • Validation errors on form submission (missing required fields, bad data).
  • System-level failures (API error, upload rejected, IC already exists).
check_circle
Success — Leave approved. Your annual leave (15–18 Apr) has been confirmed.
Green · #2DCE89 · When to use
  • Action completed successfully — confirm to the user that something worked.
  • Record saved, submitted, approved, or published.
  • Use inline (persistent) when the result is important enough to stay visible.
Placement Guide
vertical_align_top Top of form / panel
Validation errors after submission. User needs to see all issues before scrolling into fields.
wrap_text Below page title
Info or warning relevant to the whole page (e.g. read-only mode, policy notice, system degraded).
input Inline in a field group
Error or warning scoped to a specific section of a form, placed immediately before that section's fields.
filter_list Inside filter / query panel
Info explaining mode (Basic / Advanced). Error when a condition is incomplete and the filter cannot be applied.
Banner Anatomy
Padding: 10px 14px
Border-radius: --r (8px)
Border: 1px solid — tinted to match severity
Background: tinted ~10% opacity of severity colour
Icon: 15–17px Material Round · same severity colour · flex-shrink:0
Text: 12px · LH 1.55 · --text-primary body · bold coloured label prefix
Links inside banner: underline · same severity colour
Close button (optional): material-icons close · 15px · --border-strong
check_circleDo
  • Always pair colour with an icon — colour alone is not accessible.
  • Keep the message to 1–2 sentences. Link to detail if more context is needed.
  • Use Error only when the user genuinely cannot proceed.
  • Dismiss Info and Success banners automatically after 4–5 s when used as toasts.
  • Place the banner before the affected content, not after.
cancelDon't
  • Don't use Error severity for warnings — it creates alert fatigue.
  • Don't stack more than two banners on the same screen at once.
  • Don't use a banner for a single-field validation error — use the field's inline error message instead.
  • Don't use banner text in ALL CAPS — sentence case only.
